# 0010 — ACME enablement

## Context
Expose Step-CA's ACME capabilities in the UI: manage ACME provisioners and EAB
keys, and help operators onboard ACME clients (ADR-0010).

## User stories
- As an admin, I create an ACME provisioner and choose allowed challenge types.
- As an admin, when `requireEAB` is on, I generate EAB credentials for a client.
- As an operator, I copy the ACME directory URL and a ready-made client snippet.

## Functional requirements
- FR-1: Create/edit/delete ACME provisioners via the admin API, with options:
  allowed challenges (`http-01`, `dns-01`, `tls-alpn-01`) and `requireEAB`.
- FR-2: EAB management per ACME provisioner: create/list/revoke external account
  keys via `/admin/acme/eab/{provisioner}` (SDK-signed admin token). Show the EAB
  `keyID` + HMAC **once** on creation (treat as a secret).
- FR-3: Show the ACME **directory URL** per provisioner with a copy button.
- FR-4: Provide copy-paste client snippets (certbot, acme.sh, Caddy, Traefik),
  including EAB parameters when required.
- FR-5: Audit all ACME provisioner/EAB changes (0009).

## Routes / UI
- `GET /acme` (list ACME provisioners + directory URLs)
- `POST /acme/provisioners`, `POST /acme/provisioners/{name}`,
  `DELETE /acme/provisioners/{name}`
- `GET /acme/{provisioner}/eab`, `POST /acme/{provisioner}/eab`,
  `DELETE /acme/{provisioner}/eab/{keyID}`

## Acceptance criteria
- Given valid admin creds, When I create an ACME provisioner with `dns-01` and
  `requireEAB`, Then it appears with the correct options and a directory URL.
- Given an ACME provisioner with `requireEAB`, When I create an EAB key, Then the
  keyID + HMAC are shown once and listed thereafter (HMAC not re-shown).
- Given an EAB key, When I revoke it, Then it can no longer be used and is removed
  from the list.
- Given a provisioner, When I open it, Then I see working client snippets including
  EAB parameters when required.

## Tests
- ACME provisioner CRUD against a mock admin API; EAB create/list/revoke; snippet
  rendering includes correct directory URL + EAB params; audit events.

## Out of scope
- Running ACME challenges ourselves (the CA + clients do this).
