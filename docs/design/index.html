design direction: “Hairline” — crisp, border-defined operator console
What this is.

One coherent visual direction for the step-ui-ng admin UI: server-render-friendly, semantic HTML + hand-written CSS tokens (no framework), light & dark from one token set, fully usable without JavaScript. Use the theme switch (top-right) to compare light and dark — production drives it purely from prefers-color-scheme. Toggle the browser narrow to see the JS-free hamburger.

The direction in one line

Hairline treats the UI like a piece of infrastructure: 1px borders define every surface, shadows are reserved only for things that float (menus), corners are tight (6–8px), and type is IBM Plex Sans with Plex Mono for anything cryptographic. The deep-navy brand bar is a constant anchor across both themes; the accent leans cyan so it never collides with the green “valid” status. Dense tables, calm forms.

  • Trust through legibility — nothing decorative competes with the data.
  • Honest states — disabled controls always say why, with a link to fix it.
  • Secrets are write-only — shown once at creation, otherwise a set / none badge.
  • Status by label + colour — never colour alone, for accessibility.

Notes for the port to Templ + htmx

  • Tokens: css/tokens.css is the single source of truth. To go pure prefers-color-scheme, delete the two [data-theme] blocks and keep :root + the @media block.
  • No framework: all styling is in tokens.css + app.css. The only JS (js/ui.js) is progressive enhancement — copy buttons, PEM expand, theme switch.
  • JS-free: nav hamburger is a checkbox+label; dropdowns are <details>; the method-selector reveal is CSS :has(). All work with JS off.
  • htmx points are marked with a dashed cyan htmx swap target tag: test-connection result, method-selector reveal, issue success, one-time EAB HMAC.
  • Fonts are self-hosted woff2 in /fonts (OFL) — no CDN, works fully offline.
  • Secrets never become input values; rendered as set/none badges or one-time panels only.
