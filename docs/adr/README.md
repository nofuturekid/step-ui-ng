# Architecture Decision Records

We record architecturally significant decisions as **MADR** (Markdown Any Decision
Records). See <https://adr.github.io/madr/>.

- Use [`0000-template.md`](0000-template.md) for new ADRs.
- Number sequentially: `NNNN-short-title.md`.
- Status: `proposed` → `accepted` → (later) `deprecated` / `superseded by NNNN`.
- When a decision changes, write a **new** ADR that supersedes the old one; do not
  rewrite history.

The initial ADRs (0001–0010) capture the decisions made when seeding this repo.
